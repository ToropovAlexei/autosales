# Название проекта

Это комплексное приложение, состоящее из бэкенд API, фронтенд веб-приложения и Telegram-бота.

## Предварительные требования

Перед началом убедитесь, что у вас установлено следующее:
- Python 3.8+
- Node.js 18.x+
- npm или yarn
- Docker и Docker Compose

## Базы данных

Для локальной разработки используются PostgreSQL и Redis, которые можно запустить с помощью Docker Compose.

1.  **Запустите контейнеры:**
    В корневой папке проекта выполните команду:
    ```bash
    docker-compose up -d
    ```
    Эта команда запустит PostgreSQL и Redis в фоновом режиме.

## Установка и запуск

Этот проект разделен на три основных компонента: `backend`, `frontend` и `tgbot`. Следуйте приведенным ниже инструкциям для настройки каждой части.

### Бэкенд

Бэкенд представляет собой приложение FastAPI.

1.  **Перейдите в каталог бэкенда:**
    ```bash
    cd backend
    ```

2.  **Создайте файл .env:**
    Создайте файл `.env` в каталоге `backend` и заполните его необходимыми учетными данными. Вы можете использовать `.env.example` в качестве шаблона.

3.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv .venv
    source .venv/bin/activate
    ```

4.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

5.  **Инициализируйте базу данных:**
    ```bash
    python init_db.py
    ```

6.  **Запустите сервер бэкенда:**
    ```bash
    uvicorn main:app --reload
    ```
    Бэкенд будет запущен по адресу `http://127.0.0.1:8000`.

### Фронтенд

Фронтенд представляет собой приложение Next.js.

1.  **Перейдите в каталог фронтенда:**
    ```bash
    cd frontend
    ```

2.  **Установите зависимости:**
    ```bash
    npm install
    ```

3.  **Запустите сервер для разработки:**
    ```bash
    npm run dev
    ```
    Фронтенд будет запущен по адресу `http://localhost:3000`.

### Telegram-бот

Telegram-бот создан с использованием `aiogram`.

1.  **Перейдите в каталог tgbot:**
    ```bash
    cd tgbot
    ```

2.  **Создайте файл .env:**
    Создайте файл `.env` в каталоге `tgbot` и заполните его необходимыми учетными данными. Вы можете использовать `.env.example` в качестве шаблона.

3.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv .venv
    source .venv/bin/activate
    ```

4.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

5.  **Запустите монитор бота:**
    ```bash
    python monitor.py
    ```
    Монитор запустит бота и будет поддерживать его в рабочем состоянии.

## Конфигурация

- **Бэкенд:** Конфигурацию для бэкенда можно найти в `backend/config.py`. Он читает данные из переменных окружения.
- **Фронтенд:** Фронтенд подключается к API бэкенда. URL-адрес API настраивается в коде фронтенда.
- **Telegram-бот:** Конфигурация бота управляется через файл `.env` в каталоге `tgbot`.
